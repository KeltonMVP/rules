# Leader Panel

The leaders' control panel is extremely important, due to the fact that from here any leader can apply sanctions or promotions and especially dismiss certain members from their team.
The control panel page includes several sub-pages in the left menu, which help the leader monitor the activity of the player members of their faction.

## Members

From this sub-page, a leader can:

* View the date when a certain faction member last logged in.
* View the date when a player joined the faction.
* View the date when the player was last promoted.
* Promote or demote members in rank (only offline ones).
* Remove a player from the faction (only if they are offline).
* Give or delete faction warnings (FW).
* View the number of faction warnings received.

## Faction activity

By visiting this page, a leader can see the faction activity of all members. Each row in the table has a checkbox, with which the leader can declare that activity row as invalid for various reasons.

## Faction reports

This page displays to the leader an informative table that includes information about the reports of their faction members, as follows:

* Player name
* Time interval in which the activity report was recorded (one week).
* Number of accumulated points

## Pending reports

All complaints to which the leader has not yet provided a final answer are sent to this page. Here there is a simple table that includes:

* Name of the reported member
* Name of the reporting player
* Date when the complaint was registered
* A button (**View**) for the leader to reach the complaint form completed by the reporter.

**Note:** From the actual complaint page, accessed through the **View** button, the leader can issue a final response to the complaint.

## Closed reports

Here all complaints that have already received a response from the leader will appear. The table on this page displays the same details as on the **Pending reports** page.

## Pending applications

All applications that have not received a response (accepted/rejected/rejected at tests/invited) from the leader will appear on this page, being detailed:

* Name of the applicant player
* Date when the application was submitted.
* A **View** button, with which the leader reaches the application form completed by the respective applicant.

## Candidates Table

By visiting this page, the website will display a table that includes players who need to take the entry test, respectively players who have taken the entry test and the verdict given by the tester.

## Accepted applications / Rejected applications / Test failed applications / Invited applications

By visiting this page, the website will display a table that includes information about a certain player's application, an application that was marked as accepted/rejected/Failed at tests/Invited. The information in the table refers to:

* Name of the accepted player
* Date when the application was published
* Response given by the leader
* Date when the leader provided the response to the application
* A **View** button, with which the leader reaches the application form completed by the respective applicant.

## Inactivities / Resignations

By visiting this page, the website will display a table that includes active inactivities or resignations, respectively those processed.

## Blacklisted Members

By visiting this page, the website will display a table with players on the faction's blacklist.

## Application Model

From this page, the faction leader can modify the questions for the faction entry application.

* From this page, the minimum application requirements can also be modified.

## Application Search

From this page, the faction leader can search for an application by a sentence or phrase, to verify its originality.
